# Go-Logical_Expression_Parser
Парсер логических выражений

Простой интерпретатор для вычисления булевых выражений, написанный на Go.

Описание
Программа считывает объявления переменных и одно логическое выражение из стандартного ввода, затем вычисляет результат выражения.

Синтаксис
Объявления переменных:

```
имя_переменной = True;
имя_переменной = False;
```

Имена переменных должны состоять только из строчных латинских букв (a-z)
Запрещены ключевые слова: and, or, not, xor, true, false
Каждое объявление должно заканчиваться точкой с запятой

Логические выражения
Поддерживаемые операторы (в порядке приоритета):

1. not - логическое НЕ
2. and - логическое И
3. xor - исключающее ИЛИ
4. or - логическое ИЛИ

Также поддерживаются:
Булевы литералы: True, False
Переменные
Группировка с помощью скобок: ( )


Пример использования
Ввод:
```
a = True;
b = False;
c = True;
(a and b) or (not c and True)
```
Вывод:
```
False
```

Обработка ошибок:

Программа выводит [error] в случаях:
Неверный синтаксис объявлений переменных
Необъявленные переменные в выражении
Неправильный формат логического выражения
Неизвестные операторы или идентификаторы

Запуск
bash
```
go run main.go
```
Затем введите объявления переменных и выражение, завершив ввод Ctrl+D (Unix) или Ctrl+Z (Windows).

Технологии и инструменты

Go (Golang) - основной язык реализации

Стандартные библиотеки Go:

    bufio - для построчного чтения ввода
    fmt - для форматированного вывода
    os - для работы со стандартным вводом
    strings - для манипуляций со строками

Основные концепции и подходы

Лексический анализ (Lexer)

    Ручной парсинг - реализован собственный лексер без использования генераторов
    Токенизация - разбиение входной строки на токены

Синтаксический анализ (Parser)

    Рекурсивный спуск - метод парсинга выражений
    Приоритет операторов - корректная обработка порядка операций

Алгоритмы

    Префиксная нотация - обработка унарных операторов (not)
    Дерево разбора - неявное построение через рекурсивные вызовы

Особенности реализации

Типизация

    Кастомные типы - TokenType для классификации токенов
    Структуры - Token для хранения информации о токенах

Обработка ошибок

    Детальная валидация - проверка на всех этапах выполнения
    Ранний возврат - немедленное прекращение при ошибках

Структура данных

    Map для хранения переменных и ключевых слов
    Срезы для хранения последовательности токенов

Ключевые функции
```
    isValidVarName() - валидация имен переменных
    lexer() - лексический анализ
    parseExpression() и родственные - синтаксический анализ
    parseDeclaration() - разбор объявлений переменных
```
Управление состоянием

    Указатели - для отслеживания позиции в токенах при парсинге
